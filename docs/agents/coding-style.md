# Coding Style & Naming Conventions

Use TypeScript/ESM (`"type": "module"`). Indentation is 2 spaces.
Prefer descriptive, package-scoped names (e.g., `packages/eslint-plugin-format/src/rules/prettier.ts`).
Follow existing naming for tests (`*.spec.ts`) and configs (`tsconfig.*.json`).
Formatting is enforced by `dprint.json`; linting by `eslint.config.ts`.
